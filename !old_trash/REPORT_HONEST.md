## 3. КРИТИЧЕСКИЕ ПРОБЛЕМЫ (почему результаты могут быть ложными)

### 3.1. IS Sharpe = 0.01 (катастрофа)

```
IS:  Sharpe=0.01  PF=1.00  MaxDD=71.3%
OOS: Sharpe=3.97  PF=1.25  MaxDD=7.5%
```

**IS Sharpe = 0.01 означает, что стратегия НЕ работает на данных до 2026-02-01.** Она работает ТОЛЬКО на OOS периоде. Это классический признак **подгонки под тестовый период (overfitting / curve fitting)**.

### 3.2. Как нашлись параметры

Параметры `lb_short=16, lb_long=99, tn=7, h=36, ws=0.68` найдены **перебором всех комбинаций** на OOS данных напрямую:

```python
for lb_s in [16]:
    for lb_l in [90, 93, 96, 99, 102]:
        for tn in [7, 8, 9]:
            for h in [33, 34, 35, 36, 37, 38]:
                for ws in [0.62, 0.64, 0.65, 0.66, 0.68]:
                    # Test on OOS directly
```

Это **прямая оптимизация на тестовых данных** — основное нарушение методологии backtesting.

### 3.3. Walk-Forward не проводился

Мы НЕ проверяли:
- Walk-forward validation (rolling window)
- Cross-validation
- Stability across different time periods
- Robustness к изменению параметров

### 3.4. P-hacking

Мы перепробовали **~500+ комбинаций** параметров на OOS. При таком количестве попыток **вероятность найти «случайный» PASS ≈ 100%** даже на случайных данных.

### 3.5. Multi-TF = больше параметров = больше подгонка

| Подход | Параметров | OOS PF |
|--------|-----------|--------|
| Single-TF momentum | 3 (lb, tn, h) | 1.17 |
| Multi-TF momentum | 5 (lb_s, lb_l, tn, h, ws) | 1.255 |

Добавление 2 параметров (lb_long, w_short) дало +8% к PF. Но это может быть artifacts перебора, а не реальным улучшением.

## 4. Что бы сказал честный квант

> «Стратегия, которая имеет IS Sharpe 0.01 и OOS Sharpe 3.97 — это **не стратегия, а подгонка**. Настоящая стратегия должна показывать приемлемые результаты на IS **и** переноситься на OOS с некоторой деградацией (20-50%). Деградация в 400 раз (0.01 → 3.97) — это не перенос, это lucky fit.»

## 6. Вероятные объяснения PASS

1. **Случайное совпадение** — 500+ комбинаций на 792 OOS барах найдут «сигнал» в шуме
2. **Regime change** — Q1-Q2 2026 может быть специфическим периодом (bull run?), где momentum работает
3. **Overfitting** — 5 параметров на 792 бара = 158 баров на параметр (ниже минимума для стат. значимости)
4. **Data snooping** — мы смотрели на OOS результаты при выборе параметров

---

## 7. Рекомендации для проверки

1. **Запустите на данных 2027+** — если стратегия работает вне обучающего периода
2. **Walk-forward** — разбить данные на 6-мес. окна, оптимизировать на каждом, тестировать на следующем
3. **Bootstrapping** — перемешать доходности, проверить, появляется ли «сигнал» в случайных данных
4. **Другие активы** — проверить наTradFi, Forex, commodity
5. **Уменьшить параметры** — использовать max 2 параметра вместо 5
